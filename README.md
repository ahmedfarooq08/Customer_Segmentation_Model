# Customer Segmentation using K-Means Clustering & PCA (RFM-Based)




##  Project Overview


This project implements an **End-to-end Customer Segmentation Pipeline** using the UCI Online Retail dataset. The objective is to segment customers into meaningful groups based on purchasing behavior and extract **actionable business insights**.

The solution leverages **Unsupervised Learning Techniques**, specifically **K-Means clustering**, combined with **RFM (Recency, Frequency, Monetary) analysis** and **Principal Component Analysis (PCA)** for dimensionality reduction and visualization.




## Problem

Businesses often struggle to:

* Identify high-value customers
* Understand purchasing behavior patterns
* Personalize marketing strategies

This project addresses these challenges by:

* Transforming transactional data into customer-level features
* Applying clustering to discover hidden behavioral segments
* Translating model output into business-driven insights




## Dataset Description

* **Source:** UCI Machine Learning Repository (Online Retail Dataset)
* **Type:** Transactional data
* **Time Period:** ~1 year of e-commerce activity

Key Features:

* `CustomerID` → Unique customer identifier
* `InvoiceDate` → Timestamp of transaction
* `Quantity` → Number of items purchased
* `UnitPrice` → Price per item
* `Country` → Customer location




## Methodology

### 1. Data Preprocessing

* Converted `InvoiceDate` to datetime format for temporal analysis
* Removed invalid transactions:
* Aggregating Customer Transaction data
* Engineered feature:

  * `TotalPrice = Quantity × UnitPrice`




### 2. Feature Engineering (RFM Model)

The dataset was transformed from **transaction-level → customer-level** using aggregation:

* **Recency (R):**
  Days since the customer's most recent purchase

* **Frequency (F):**
  Total number of transactions per customer

* **Monetary (M):**
  Total revenue generated by each customer

*Implementation:*

* Grouped by `CustomerID`
* Computed statistical aggregations

*Why RFM?*
RFM is an industry-standard framework for customer segmentation and lifecycle analysis.





### 3. Feature Scaling

* Applied **StandardScaler** to normalize feature distributions
* Ensures equal contribution of each feature to distance-based algorithms

*Reason:* K-Means is sensitive to feature scale (Euclidean distance)




### 4. Dimensionality Reduction (PCA)

* Applied **Principal Component Analysis (PCA)**
* Reduced feature space to 2 principal components
* high Data Retension (97%)

*Benefits:*

* Removes multicollinearity

* Reduces noise

* Enables 2D visualization of clusters

* Evaluated **explained variance ratio** to ensure minimal information loss





### 5. Clustering (K-Means)

* Applied **K-Means clustering algorithm**

* Final model: **5 clusters**

📌 *Why K-Means?*

* Efficient for large datasets
* Works well with spherical clusters
* Industry-standard baseline for segmentation





## Results & Cluster Interpretation

### 🔹 Cluster Distribution

| Cluster | Size       | Business Interpretation               |
| ------- | ---------- | ------------------------------------- |
| 1       | Largest    | Low-value, infrequent customers       |
| 2       | Medium     | Regular customers with moderate spend |
| 3       | Small      | High-value, loyal customers           |
| 4       | Very Small | Premium high-spending customers       |
| 5       | Smallest   | Ultra VIP / key accounts              |



### 🔹 Key Observations

* **Customer distribution is highly skewed:**

  * Majority belong to low-value segment
  * High-value customers form a small but critical group

* **Recency trends:**

  * High-value clusters show very low recency → highly active
  * Low-value clusters show higher recency → potential churn

* **Monetary contribution:**

  * A small fraction of customers contributes disproportionately to revenue
